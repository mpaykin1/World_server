# Правила игры — Мир Шарабас (World-Sharabass)

Музыкальный 3D мир, где ты ходишь, слушаешь музыку и строишь объекты.

---

## 1. Мир и окружение

- **Графика**: SDF raymarching (WebGL2) — деформирующийся центральный blob, pillar, orb, groundOrb, torus, ground
- **Размер мира**: ~60×60 метров (сетка в центре, персонаж ходит по плоскости)
- **Центральный объект**: пульсирует от музыки и касаний. Деформация сцены: скрутка, волны, изменение масштаба
- **Погода**: ясно → облака → дождь → молнии → снег → дым. Меняется от Y-касания на телефоне или от времени на ПК. Синхронизируется между игроками
- **Освещение**: ambient + diffuse + specular + Fresnel + glow. Меняется от погоды (облака затемняют)

---

## 2. Персонаж

- **Вид**: третье лицо (камера сзади-сверху, distance=6, height=4)
- **Q** — переключение на первое лицо (distance=0.15, height=1.6, персонаж скрывается)
- **Модель THREE.js**: капсула-тело + голова с лицом (canvas-текстура) + руки/ноги коробками + синий диск под ногами
- **Анимация**: руки/ноги качаются при ходьбе, затухают при остановке
- **Движение**: frame-independent (скорость 0.12 × dt)
- **Поворот**: мышь drag (PC), 2 пальца drag (touch)

### Управление

| Действие | PC | Touch |
|----------|----|-------|
| Ходьба | WASD / стрелки | 1 палец (drag) |
| Поворот камеры | Мышь (drag) | 2 пальца (drag) |
| Вид от 1-го лица | Q | — |
| Режим стройки | B (или кнопка 🚧) | кнопка 🚧 |
| Поставить объект | Space / Enter / ЛКМ | тап |
| Размер объекта | Q (меньше) / E (больше) | — |
| Тип объекта | кнопка 🔵 Шар / 🧊 Куб | кнопка |
| Удалить последний | 🗑️ | 🗑️ |
| Музыка | Любая клавиша / клик | Тап (2-й и далее) |
| Чат | Enter | — |

## 3. Музыка (Web Audio API)

- **Запуск**: первый клик/клавиша/тап (обход автоплей-блокировки браузера)
- **12 аккордов** (C → Dmin7), циклически сменяют друг друга
- **4 слоя**:
  - Аккорд: 4 осциллятора + реверберация
  - Бас: корень/квинта через фильтр
  - Мелодия: случайные ноты, темп от скорости тапа
  - Build boost: всплеск после установки объекта (затухает ~2с)
- **Громкость и сложность** зависят от очков силы (см. раздел 6)
- **Звук установки**: C5-E5-G5 мажорный аккорд (при каждом placement)
- **Звук молнии**: шум с полосовым фильтром (2 типа: cloud/branching)
- **Build boost**: при установке объекта временно повышает bassLevel (+0.2), harmonyComplexity (+0.4), melodyActivity (+0.3)

---

## 4. Стройка

- **Вход**: B (PC) / кнопка 🚧 (touch)
- **Выход**: автоматически после установки объекта
- **Сетка**: snap `Math.round(v/2)*2` (ячейки 2×2)
- **Призрак**: полупрозрачный шар/куб на ближайшей ячейке в 2м перед персонажем
- **Типы**: 🔵 Шар (type=0), 🧊 Куб (type=1)
- **Размер**: Q (уменьшить) / E (увеличить) — без ограничений
- **Установка**: Space / Enter / ЛКМ / тап
- **Удаление**: 🗑️ удаляет последний собственный объект
- **Материал объектов**: металлический (темная сталь 0.18/0.20/0.22, specular 64, Fresnel отражения)
- **Максимум объектов**: 50 на весь мир (`MAX_OBJECTS`)
- **Деформация от музыки**: bass → пульсация размера, harmony → скрутка, melody → волны поверхности
- **Uniform-массивы**: обновляются только при `init/place/remove`, не каждый кадр

---

## 5. Сеть и мультиплеер

- **Все объекты** видны всем игрокам в реальном времени
- **Другие игроки** видны как такие же персонажи (stick figures), позиция обновляется из события `sharabass:fly`
- **Погода** синхронизируется: один игрок меняет — всем видно
- **Чат**: Enter → глобальный чат с историей
- **Авторизация**: логин/регистрация через `#authBox` (JWT token)

### WebSocket события

| Событие | Направление | Описание |
|---------|-------------|----------|
| `sharabass:join` | client → server | Вход в мир (обязательно! без него всё игнорируется) |
| `sharabass:fly` | client → server | Позиция камеры (троттлинг 80мс) |
| `sharabass:place` | client → server | Установка объекта |
| `sharabass:remove` | client → server | Удаление своего объекта |
| `sharabass:weather` | client → server | Смена погоды |
| `sharabass:init` | server → client | Инициализация (selfId, objects, weather, players) |
| `sharabass:object:placed` | server → broadcast | Новый объект |
| `sharabass:object:removed` | server → broadcast | Объект удалён |
| `sharabass:weather` | server → broadcast | Синхронизация погоды |
| `sharabass:players` | server → broadcast | Список игроков |

## 6. Очки силы (⚡)

Начисляются в реальном времени, пока ты в игре.

**Формула**: `ppRate = (prox×0.5 + sym×0.3 + sizeBonus×0.2) × dt`

| Фактор | Что даёт | Расчёт |
|--------|----------|--------|
| **Близость к центру** | prox = 1 − clamp(dist/30, 0, 1) | Чем ближе к центру (0,0), тем больше очков |
| **Симметрия архитектуры** | sym = pairs / totalOwnObjects | Объекты зеркально относительно центра (x↔−x, z↔−z) в пределах 0.5м |
| **Размер объектов** | sizeBonus = min(1, sum(sizes)/20) | Сумма размеров всех твоих объектов |

**Влияние на музыку** (добавляется к bass/harmony/melody):
- Близость к центру: до +0.08/0.10/0.12
- Симметрия: до +0.12/0.15/0.18
- Размер: до +0.06/0.08/0.10

**Отображение**: ⚡N в левом верхнем углу экрана (рядом с расстоянием от центра).

## 7. UI и интерфейс

- **Масштаб**: все панели уменьшены (scale 0.6) и полупрозрачны (opacity 0.7)
- **Закрытие**: × на каждой панели — скрыть. После перезагрузки всё восстанавливается
- **Верхний левый угол**: подсказка по управлению + расстояние от центра + очки силы ⚡
- **Верхний правый угол**: список игроков онлайн, текущая погода
- **Внизу (центр)**: кнопки 🚧 Строить / 🔵 Шар / 🧊 Куб / 🗑️
- **Полоска размера**: отображается над кнопками при стройке
- **Диагностика**: `data-sharabass-*` атрибуты на body (renderer, gpu-tier, perf-mode, fps, frame-ms, render-quality)

---

## 8. Производительность

- **Адаптивное качество**: автоопределение GPU tier (low / normal)
- **GPU tiers**:
  - **normal**: более высокое качество (больше ray steps, FBM octaves, smoke samples)
  - **low**: сниженные лимиты для старых GPU (Intel HD 4000 и т.п.)
- **Режимы через URL**:
  - `?perf=max` — максимальная скорость (min ray steps, min quality 0.22)
  - `?perf=balanced` — компромисс качества и скорости (по умолчанию)
  - `?perf=quality` — максимальное качество
- **Движение frame-independent**: `WALK_SPEED * dt` (не зависит от FPS)
- **На low-power GPU** режим по умолчанию `max`
